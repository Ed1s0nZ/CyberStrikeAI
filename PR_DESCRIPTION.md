# 固定对话列表排序为创建时间

## 变更内容

- 将主对话列表、未分组对话列表、WebShell AI 对话列表的后端排序从 `updated_at DESC` 改为 `created_at DESC`。
- 将分组内对话排序改为「分组置顶优先，然后按创建时间倒序」，避免继续对话后位置变化。
- 将项目关联对话列表排序改为按创建时间倒序。
- 将 Web 前端对话列表的二次排序、时间分组、列表时间显示从 `updatedAt` 切换为 `createdAt`。
- 将项目关联对话和 WebShell AI 对话列表的时间显示优先使用创建时间。

## 背景与目的

之前对话列表按 `updated_at DESC` 排序，用户在旧对话中继续发送消息后，该对话会因为更新时间变化跳到列表顶部。现在改为按创建时间排序，使对话创建后的列表位置保持稳定，不再随后续聊天变化。

## 影响范围

- 普通对话侧栏列表
- 未分组「最近对话」列表
- 分组内对话列表
- 项目关联对话列表
- WebShell AI 对话列表

消息气泡顺序没有改变，仍按消息时间/插入顺序展示。`updated_at` 字段仍会被更新，只是不再参与这些对话列表的排序和主要时间展示。
